# ЧТЗ Этап 5: Слой Эталонов и Нормативов (Reference)

## 1. Задача
Вынос всех бизнес-целей, нормативов, бенчмарков и весовых коэффициентов из кода визуализации и трансформаций в строгие справочные таблицы БД.

## 2. Исходное состояние
- Пороги для "светофора" (например, < 50% = красный) захардкожены в `generate_heatmap.py`.
- Нормативы обеспеченности захардкожены в `dashboards/`.
- Файлы конфигураций лежат в `ingestion/normalization_config.yaml` или `data/`.

## 3. Целевое состояние
- Создается набор статических узлов `e_*` (Etalons).
- Каждый такой узел читает статический конфиг (YAML/JSON/CSV) из репозитория и возвращает `pd.DataFrame`, который Dagster материализует в БД.
- Дашборды и расчетные метрики (Слой 6) будут брать нормативы только из этих таблиц.

## 4. Требования к реализации

### 4.1. Извлечение нормативов (Config to Table)
Проанализировать логику `generate_heatmap.py`, `dormitory_dashboard.py` и скриптов в папке `transformations/gold_*.py`. 
Найти все места, где используются константы для сравнения (например, целевой % охвата, нормативы квадратных метров на студента).

### 4.2. Формат данных
Узлы эталонов должны возвращать структурированные датафреймы. 
Пример для `e_color_thresholds`: колонки `indicator_name`, `red_threshold`, `yellow_threshold`, `green_threshold`.
Пример для `e_target_coverage`: колонки `year`, `target_percent`.

## 5. Требуемые ассеты (dagster_project/assets/5_etalon_assets.py)
Группа `5_etalon`:
- `e_color_thresholds` (Пороги для раскраски heatmaps)
- `e_business_targets` (Целевые плановые показатели по годам)
*(Агент может расширить этот список, если найдет другие хардкод-константы в бизнес-логике)*

**Конфигурация:** Настроить IO Manager на сохранение в схему `reference`.
