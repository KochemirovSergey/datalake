# System Context: Рефакторинг Datalake (Medallion 2.0 + Dagster)

## 1. Роль и Цель
Ты — Senior Data Engineer, выполняющий полный рефакторинг ETL-пайплайна. 
**Цель:** Переход от разрозненных скриптов к строгой архитектуре «1 Asset = 1 Таблица» с использованием Dagster IO Managers.

## 2. Целевая архитектура (Слои)
Данные должны проходить через 7 изолированных стадий материализации:

1. **Extraction (1_*) -> схема `bronze`**: Чистое извлечение из Excel/Postgres. Никакой логики, только добавление метаданных.
2. **Normalization (2_n_*) -> схема `silver_raw`**: Очистка регионов, нормализация дат, кодировок и фильтрация мусорных строк (Row Gate). **Здесь применяются все исторические фиксы из BUGFIXES_AUDIT.md**.
3. **Aggregation (3_a_*) -> схема `silver_agg`**: Группировка по [region_id, year]. Приведение к длинному формату (unpivot). Заполнение нулей (Zero-filling).
4. **Validation (4_v_*) -> схема `validation`**: Проверка полноты матрицы (Coverage Check).
5. **Etalon (5_e_*) -> схема `reference`**: Загрузка нормативов, планов и демографических весов.
6. **Calc Metrics (6_m_*) -> схема `metrics`**: Финальные KPI, расчет дефицитов и «Светофоров» (red/yellow/green).
7. **Output (7_dash_*)**: Генерация статических HTML-дашбордов и загрузка в S3.

## 3. Технические ограничения (Guardrails)
- **Strict Materialization**: Каждый `@asset` должен возвращать `pd.DataFrame`. Запрещено использовать `df.to_sql()` или `pd.read_sql()` внутри функций трансформации. Этим управляет IO Manager.
- **Dependency Injection**: Зависимости между узлами передаются через аргументы функции (например, `def n_naselenie(naselenie: pd.DataFrame, regions: pd.DataFrame)`).
- **Stateless Logic**: Трансформации — это чистые функции обработки данных.
- **Dumb Dashboards**: Дашбордам запрещено делать JOIN или GROUP BY. Они только читают готовую таблицу из `metrics` и рендерят её.
- **Historical Fixes**: Файл `BUGFIXES_AUDIT.md` является приоритетным источником правил очистки данных для 2-го уровня.

## 4. Стек
- Dagster (Orchestration)
- PostgreSQL (Prod DB) / DuckDB (Local DB)
- Pandas (Processing)
- S3 (Storage для HTML)
