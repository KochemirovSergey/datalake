## 🚨 Критические пропуски

### 1️⃣ `n_ped_doshkolka` — Педагоги дошколки (T_Norm1) ✅ РЕАЛИЗОВАН
- **Диаграмма**: T_Excel → T_Norm1 → Val_T_Cov (прямо на Слой 4)
- **Схема выхода**: `[region_code, year, edu_level_code='1.1', ped_category, metric_name, stazh_type, value]`
- **Листы**: ОКАТО-коды из `data/OKATO_territories.csv`; агрегаты (РФ + 8 ФО) пропускаются
- **Метрики**: возраст (10 групп), образование (2), ставки (2, только 2022+), стаж (6×2: общий + пед.)
- **Реализован**: `2_normalization_assets.py`, строка ~1082

### 2️⃣ `n_ped_pk` — Педагоги ПК (T_Norm5)
- **Диаграмма**: T_PG4 → T_Norm5 (в диаграмме нет выходящих связей от T_Norm5!)
- **L1-источник**: `pk_1_raw` — 🟡 сырой ассет, все листы. Нужна фильтрация на Слое 2.
- **Проблема**: Нет нормализующего ассета. Схема `pk_1_raw` не ясна без анализа источника.
- **Приоритет**: 🟡 Средний — T_Norm5 в диаграмме не имеет выходящих связей, значит до Слоя 4 не доходит.

### 3️⃣ `n_ped_spo` — Педагоги СПО (T_Norm3)
- **Диаграмма**: T_PG2 → T_Norm3 → T_Agg2
- **L1-источник**: ❌ `ped_spo` отсутствует в L1
- **Проблема**: Двойной пропуск (нет ни L1, ни L2). Неизвестно, существует ли таблица в Postgres.
- **Приоритет**: 🔴 Высокий — блокирует T_Agg2 (L3) и далее T_Sep3, T_Sep4 (Слои разделения).

### 4️⃣ `n_ped_vpo` — Педагоги ВПО (T_Norm4)
- **Диаграмма**: T_PG3 → T_Norm4 → T_Agg3
- **L1-источник**: ❌ `ped_vpo` отсутствует в L1
- **Проблема**: Двойной пропуск. Неизвестно, существует ли таблица в Postgres.
- **Приоритет**: 🔴 Высокий — блокирует T_Agg3 (L3).

### 5️⃣ `n_obshagi_spo` — Общаги СПО (D_Norm1)
- **Диаграмма**: D_PG1 → D_Norm1 → D_Agg1 → Val_D_Cov
- **L1-источник**: ❌ Нет
- **Текущее состояние**: `n_obshagi_spo` существует как заглушка `UNIMPLEMENTED`, возвращает пустой DataFrame.
- **Приоритет**: 🟡 Средний — заглушка не ломает граф, но блокирует KPI общаг СПО.

## ✅ Рекомендации

### Немедленные действия (L1-источник есть):
**A.** Создать `n_ped_doshkolka` — нормализатор педагогов дошколки из `ped_doshkolka`.
- Входные колонки: `_ped_category` (возраст/стаж/образование/ставки) + данные по столбцам
- Выход: `[region_code, year, edu_level_code='1.1', ped_category, ...]`
- Согласно диаграмме — идёт напрямую на Слой 4 (нет age-нормализации для педагогов).

**B.** Создать `n_ped_pk` — нормализатор педагогов ПК из `pk_1_raw`.
- Требует анализа структуры `pk_1_raw` (листы, схема).
- T_Norm5 в диаграмме — тупик (нет связи с L3), уточнить назначение.

### Уточнить у заказчика:
**C.** Существуют ли в Postgres таблицы педагогов СПО и ВПО?
- Если да → создать L1-ассеты `ped_spo`, `ped_vpo`, затем `n_ped_spo`, `n_ped_vpo`.
- Если нет → убрать T_PG2, T_PG3 из диаграммы.

**D.** Существует ли в Postgres таблица общаг СПО?
- Если да → реализовать L1 + снять заглушку `n_obshagi_spo`.
- Если нет → убрать D_PG1 из диаграммы, оставить заглушку.

### Дополнить диаграмму:
**E.** Прописать маршрут `S_Age5` (ПК-обучающиеся) → нет ни связи на L3, ни на L4.
**F.** Прописать маршрут `T_Norm5` (ПК-педагоги) → нет выходящих связей.
